Настройка php.ini

[Xdebug]
zend_extension = "%sprogdir%/modules/php/%phpdriver%/ext/php_xdebug.dll"

;включение
xdebug.remote_enable = 1

;протокол
xdebug.remote_handler = "dbgp"

;сервер
xdebug.remote_host = "localhost"

;порт
xdebug.remote_port = 9000

;режим дебага
xdebug.remote_mode = "req"

;не стартовать отладку автоматически
xdebug.remote_autostart = 0

;ключ IDE
xdebug.idekey = "PHPSTORM"

;ссылка для ошибок для открытия в phpstorm
xdebug.file_link_format = "phpstorm://open?%f:%l"


;не включать трассировку
xdebug.auto_trace = 0

;включать трассировку на лету из IDE
xdebug.trace_enable_trigger = 1

;формат логов 1 файл на 1 url
xdebug.trace_output_name = "trace_%H_%R"

;папка логов
xdebug.trace_output_dir = "%sprogdir%/userdata/temp/xdebug/"


;не включать профайлер
xdebug.profiler_enable = 0

;включать профайлер на лету из IDE
xdebug.profiler_enable_trigger = 1

;формат логов 1 файл на 1 url
xdebug.profiler_output_name = "profiler_%H_%R"

;папка логов
xdebug.profiler_output_dir = "%sprogdir%/userdata/temp/xdebug/"
-------------------------------------------------

Отладка в PhpStorm

Поставить плагин Xdebug helper
https://chrome.google.com/webstore/detail/xdebug-helper/eadndfjplgieldjbigjakmdgkmoaaaoc
В настройках плагина прописать IDE key PHPSTORM

В меню OpenServer сделать закладку на Phpstorm и оттуда запускать.

Просмотр логов профилирования
Tools -> Analyze Xdebug Profiler Snapshot

Step Over
выполнение кода построчно, без заходов в функции

Step Into
выполнение кода построчно, с заходом в функции

Step Out
выход из текущей функции, к строке сразу после неё

Run to Cursor
выполнить до курсора

Resume Program
переход к следующему брекпоинту и потом стоп
